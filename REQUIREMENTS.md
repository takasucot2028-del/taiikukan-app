# 体育館月間使用計画　予約管理アプリ 要件定義

## プロジェクト概要

鷹栖中学校の部活動地域移行にともなう体育館施設使用計画を管理するWebアプリケーション。  
指導者（各地域クラブ担当者）が占有予約を申請・確認し、管理者が月間予定表を生成・確定・Excel出力できる。  
データ基盤はGoogleスプレッドシートとし、Google Apps Script（GAS）をAPIサーバー代わりに使用する。

---

## 技術スタック

| 領域 | 技術 | 理由 |
|------|------|------|
| フロントエンド | React + Vite + TypeScript | スマホ対応・軽量SPA |
| スタイル | Tailwind CSS | モバイルファーストUI |
| ホスティング | Cloudflare Pages（無料） | 費用0円・独自ドメイン不要 |
| バックエンド/DB | Google Sheets + Google Apps Script | 費用0円・既存Excelデータ移行可 |
| 認証 | なし（URLアクセスのみ） | 要件通り |
| 通知 | Web Push API + Service Worker | ブラウザプッシュ通知 |
| Excel出力 | SheetJS（クライアントサイド） | 費用0円 |

## ユーザー種別

### 指導者（一般ユーザー）
- 人数：50名
- メインデバイス：スマートフォン
- アクセス方法：URLを知っていればログイン不要でアクセス可
- 自分のクラブ名をアプリ内で選択して使用する（セッション保持）

### 管理者
- 人数：1名
- アクセス方法：管理者専用URL（`/admin`）＋ PINコード認証（4〜6桁）
- 管理者PINはGoogleスプレッドシートの設定シートで管理

## 画面構成

### 指導者アプリ（`/`）

#### 1. クラブ選択画面
- 初回アクセス時に自分のクラブ名を選択
- 選択内容はlocalStorageに保存（次回以降スキップ）
- クラブ一覧はGoogleスプレッドシートの「クラブ一覧」シートから取得

#### 2. ホーム画面（カレンダー）
- **月間カレンダー表示**と**リスト表示**をタブで切り替え
- デフォルト：全クラブの予定を表示
- フィルター：自分のクラブだけに絞り込み可能
- 色分け：土曜（水色）・日曜（オレンジ）・祝日（薄赤）・学校行事（水色系）・長期休業（緑）
- 占有予約がある日は施設名アイコンをバッジ表示
- 日付をタップ → その日の詳細モーダル表示

#### 3. 日別詳細モーダル
- 時間帯（8:00〜11:00 / 11:00〜14:00 / 14:00〜17:00 / 16:00〜18:00）ごとの使用クラブ一覧
- 占有予約がある場合は「【大会占有】」表示
- 「＋ 占有予約を申請する」ボタン

#### 4. 占有予約申請画面
- 入力項目：
  - 申請クラブ名（自動入力・変更不可）
  - 日付（カレンダーピッカー）
  - 時間帯（ドロップダウン）：8:00〜11:00 / 11:00〜14:00 / 14:00〜17:00 / 16:00〜18:00 / 終日
  - 占有施設（ドロップダウン）：第1体育館 / 第2体育館 / 総合体育館 / 第1・第2体育館 / 全施設
  - 占有内容（テキスト入力）：大会名・練習試合相手など
  - 管理者へのコメント（テキスト入力・任意）
- 送信ボタン押下 → Googleスプレッドシートの「予約申請」シートに書き込み
- 翌月20日を過ぎた申請は警告メッセージを表示（送信は可能）

#### 5. 自分の予約一覧画面
- 自分のクラブが申請した予約の一覧
- 各予約に「編集」「取り消し」ボタン
- ステータス表示：申請中 / 確定 / 却下
- 確定済みの予約は編集・取り消し不可（管理者に要連絡のメッセージを表示）

---

### 管理者アプリ（`/admin`）

#### 1. 管理者ログイン画面
- PINコード入力（4〜6桁）
- sessionStorageに認証状態を保存

#### 2. ダッシュボード
- 今月・来月の予約申請件数サマリー
- 未処理の申請件数バッジ
- 締め切りまでの残日数表示
- クイックリンク：予定表生成・設定・通知送信

#### 3. 予約申請管理
- 申請一覧テーブル（日付・クラブ・施設・時間帯・内容・コメント・ステータス）
- ステータス変更：申請中 → 確定 / 却下
- 承認フローなし（管理者が直接ステータスを変更）
- フィルター：月・クラブ・ステータス・施設で絞り込み
- 競合チェック：同日・同施設・同時間帯に複数申請がある場合は赤くハイライト

#### 4. 月間予定表生成・編集
- 対象年月を選択
- 「予定表を生成」ボタン：
  - 祝日・学校行事を自動反映
  - 平日固定スケジュールを適用
  - 土曜・日曜ローテーションを自動計算
  - 確定済み占有予約を反映
  - NexusBC固定ルール（日曜14:00〜17:00）を適用
- 生成されたExcel予定表と同じ形式のプレビュー表示
- セルを直接クリックして手動編集可能
- 変更履歴メモの入力欄
- 「予定表を確定」ボタン（毎月21日以降に有効）
- 「共有用Excelを出力」ボタン：値のみのxlsxをダウンロード

#### 5. 設定管理
- **クラブ一覧**：クラブ名の追加・編集・削除
- **平日固定スケジュール**：月〜金の各施設使用クラブ設定
- **夏期ローテーション**（5〜10月）：土曜3パターン・日曜3パターン × 3時間帯
- **冬期ローテーション**（11〜4月）：土曜6パターン・日曜6パターン × 3時間帯
- **祝日リスト**：年間祝日の追加・編集
- **学校行事リスト**：学校行事の追加・編集
- **管理者PINコード変更**
- すべての設定変更はGoogleスプレッドシートの「設定」シートにリアルタイム保存

#### 6. 通知管理
- **リマインド通知**：毎月15日に自動送信（「翌月予定表の申請締め切りは20日です」）
- **確定通知**：管理者が手動で「確定通知を送信」ボタンを押したとき
- 通知履歴の確認
- 通知の有効登録数（プッシュ通知を許可した端末数）の表示

#### 7. 変更履歴・ログ
- 誰がいつ何を申請・変更・削除したかのログ一覧
- Googleスプレッドシートの「ログ」シートに自動記録

## Googleスプレッドシート構成

### シート一覧

| シート名 | 用途 |
|----------|------|
| 設定 | 既存Excelの設定シートをそのまま移行（クラブ・ローテーション・祝日・学校行事） |

| 予約申請 | 指導者からの占有予約申請データ |
| ログ | 操作ログ |
| プッシュ通知登録 | 端末のプッシュ通知subscription情報 |

| 列 | 項目 | 型 |
|----|------|----|
| A | 申請ID | 文字列（UUID） |
| B | 申請日時 | 日時 |
| C | クラブ名 | 文字列 |
| D | 対象日 | 日付 |
| E | 時間帯 | 文字列 |
| F | 占有施設 | 文字列 |
| G | 占有内容 | 文字列 |

## Google Apps Script API エンドポイント

GASをWebアプリとしてデプロイし、フロントエンドからfetchで呼び出す。

```
GET  ?action=getConfig          → 設定シートの全データ取得
GET  ?action=getSchedule&year=YYYY&month=MM → 月次予定表データ取得
GET  ?action=getReservations&year=YYYY&month=MM → 予約申請一覧取得
POST action=addReservation      → 予約申請追加
POST action=updateReservation   → 予約申請更新（編集・取り消し）
POST action=updateStatus        → ステータス変更（管理者）
POST action=saveSchedule        → 予定表保存（管理者）
POST action=saveConfig          → 設定保存（管理者）
POST action=registerPush        → プッシュ通知登録
POST action=sendNotification    → 通知送信（管理者）
GET  ?action=getLogs            → ログ取得（管理者）
```

## 通知仕様

### Web Push通知
- フロントエンドにService Workerを実装
- 指導者がアプリを初回起動時にプッシュ通知の許可を求める
- subscriptionオブジェクトをGASに送信してスプレッドシートに保存
- GASのTime-drivenトリガーで毎月15日にリマインド通知を自動送信
- 管理者が「確定通知を送信」ボタンを押すと全登録端末に送信

### 通知内容
| タイミング | タイトル | 本文 |
|-----------|----------|------|
| 毎月15日自動 | 「体育館予約 締め切り通知」 | 「翌月分の占有予約申請の締め切りは今月20日です。申請漏れがないかご確認ください。」 |
| 管理者手動（確定時） | 「体育館予約 XX月予定表確定」 | 「XX月の体育館使用予定表が確定しました。アプリでご確認ください。」 |

## 月間予定表の自動生成ロジック

既存Excelのロジックをそのまま移植する。

1. 対象月の日付を生成
2. 祝日リスト・学校行事リストと照合して区分を判定
   - 優先順位：学校行事 ＞ 祝日 ＞ 土曜 ＞ 日曜 ＞ 平日
   - 学校行事の平日は平日扱い（ローテーションなし）
3. 平日 → 設定の平日固定スケジュールを適用（16:00〜18:00のみ）
4. 土曜 → 土曜ローテーションを適用（開始番号は管理者が設定）
5. 日曜・祝日 → 日曜ローテーションを適用
   - 14:00〜17:00 は第1全面・第2全面に「NexusBC」固定
6. 長期休業の平日 → 管理者が手動で区分を「長期休業」に変更 → 日曜ローテーション適用
7. 確定済みの占有予約がある場合 → 該当施設・時間帯を「【大会占有】」で上書き

## 注意事項・制約

- GAS WebアプリはCORSの制限があるため、フロントエンドからのfetchはGAS側で`Access-Control-Allow-Origin`を設定する
- GoogleスプレッドシートAPIの無料枠：読み取り300リクエスト/分、書き込み60リクエスト/分
- Web Push通知はHTTPS必須（Cloudflare Pagesは標準でHTTPS）
- iOSのSafariはWeb Push対応がiOS 16.4以降のため、古い端末では通知が届かない場合がある
- 管理者PIN認証はsessionStorageを使用するため、ブラウザを閉じると再ログインが必要
- ログインなしのURLアクセスのため、URLの共有先には注意が必要（セキュリティは最低限）
